# BLESC Webサイト：Figmaアセット／パララックス設計計画

更新日: 2026-07-14  
対象: `BLESC-website` の公開サイト（`blesc.jp`）

## 目的

参考画像のように、ファーストビューからスクロールするたびに空気感と奥行きが切り替わる、印象的なサイトをつくる。

BLESCでは高級物件の写真の代わりに、**「声にならないSOSを、孤立する前に支援へつなぐ」**という体験を、静かで信頼できる映像表現で伝える。対象が学校・生徒であるため、実在の未成年、相談ログ、特定可能な学校情報はアセットに使わない。

## 現状から引き継ぐブランド基盤

| 項目 | 現行サイトから引き継ぐ内容 |
| --- | --- |
| 基調色 | ブルー `hsl(215, 100%, 46%)`、白、深いネイビー系文字 |
| 文字 | Inter と Noto Sans JP。英語見出し＋日本語の短いメッセージ |
| モチーフ | `public/assets/flower.png` の花。気づき・つながりの象徴 |
| 中心メッセージ | 「Hearing the unspoken. Preventing the unseen.」「生徒のSOSを可視化する。」 |
| 守るべき約束 | 会話ログは教員に共有しない。非診断・支援の判断材料であり、治療や危機対応の代替ではない。 |

## クリエイティブ方針

### 目指す印象

- ミニマルで上質。ただし冷たい医療機器のようにはしない。
- 目に見えない心の変化が、光・粒子・線・花の抽象モチーフとして少しずつ現れる。
- スクロールで「静かな教室」から「対話」「匿名化されたシグナル」「支援者の見える要点」へ進む。
- 青は常時強く使わず、重要な変化や行動を示す光として使う。

### 避ける表現

- 生徒の顔、制服、校名、チャット原文、個人を特定できる画面。
- 苦痛や自傷を直接連想させるショッキングな画像。
- 「診断する」「完全に防ぐ」と受け取られる断定表現。
- リスクを赤い警報だけで煽るUI。高リスク表現は、警告色よりも支援につながる優先度表示として設計する。

## Figmaで作るページとアセット

### 1. `00 Foundations` — 再利用できる基盤

- Color variables: `Brand/Blue`, `Ink/900`, `Ink/600`, `Canvas/White`, `Surface/Mist`, `Signal/Low`, `Signal/Medium`, `Signal/High`
- Typography styles: Display English, Japanese H1/H2/H3, Body, Caption, Eyebrow
- Effects: 背景光彩、ガラス面のぼかし、低コントラストの影、グリッド
- Spacing / radius variables: 4, 8, 12, 16, 24, 32, 48, 80, 120 / 16, 24, 32, pill
- Motion spec tokens: `enter`, `drift`, `reveal`, `parallax-slow`, `parallax-fast`

### 2. `01 Brand Assets` — ロゴと抽象ビジュアル

| アセット | 用途 | Figmaでの作り方 |
| --- | --- | --- |
| 花ロゴのライト／ダーク版 | ヘッダー、導入アニメーション、OG画像 | 既存PNGを配置し、必要ならSVG原本も参照して書き出し用フレームを作成 |
| `Signal Bloom` | Hero背景、セクション遷移 | 花の輪郭を起点に、淡い青の発光粒子と同心円を重ねたベクター／ラスターレイヤー |
| `Quiet Grid` | Hero背景・長いスクロール背景 | 64px基準の細いグリッド。画面端でフェードアウトするパターン |
| `Privacy Veil` | AI解析・プライバシー説明 | 会話を読めない形に抽象化する、ぼかした短冊・点群・鍵のモチーフ |
| `Support Constellation` | 教員への要点レポート | 個人を表さないノードと接続線。明るさで優先度を示す |

### 3. `02 Landing / Desktop 1440` — スクロール体験の主要フレーム

画面は一枚の長い静的デザインではなく、実装・検証しやすいように以下のシーン別フレームを作る。各シーンに「開始」「中間」「終了」の3状態を置き、パララックス量・不透明度・スケールを注記する。

| シーン | 伝えること | 主要アセット | スクロール演出 |
| --- | --- | --- | --- |
| Intro / Hero | 見えないSOSに気づく | `Signal Bloom`、花、`Quiet Grid` | 背景のグリッドは遅く、花と光は少し速く動く。CTAは前景に固定感を出す。 |
| Reality | 学校が直面する現実 | 数字、余白、薄い光の残像 | 数字と課題カードが下から静かに現れ、装飾文字だけが横へ流れる。 |
| How it works | 生徒→AI→教員の安全な流れ | `Privacy Veil`、3ノードのパイプライン | 左のパイプラインをstickyにし、右の説明に合わせて光がノード間を移動。 |
| Product | 生徒画面と教員画面の違い | 架空・匿名のチャットUI、要点レポートUI | 二つのUIカードを奥行き違いで移動。会話内容は常にダミー文のみ。 |
| Why BLESC | 4つの価値 | 4枚のベネフィットカード | 背景光が各カードのアクセントに変化。ホバー時の浮上は控えめ。 |
| Technology | 研究に根ざした構造 | `Support Constellation`、知識グラフ | ノードと線が段階的に結ばれる。因果関係は概念例として明示する。 |
| Philosophy / Team / Process | 信頼・伴走・導入 | 花の余韻、抽象人物シルエット、プロセス線 | 前半より運動量を落とし、読みやすさを優先。 |
| Contact / Footer | 行動への導線 | 発光を抑えたフォーム背景、花の小モチーフ | パララックスを止め、入力と可読性を最優先。 |

### 4. `03 Components` — 実装に渡せる部品

- Header: 通常／スクロール後／モバイルメニュー開閉
- Button: Primary / Secondary / Text、通常・hover・focus・disabled
- Section eyebrow、見出し、本文、数値統計
- Glass card / benefit card / process step
- Pipeline node: 生徒、AI解析、教員
- Product mock: 生徒チャット、教員向け匿名要点レポート
- Signal badge: 低・中・要確認（機械的な「診断」ラベルにはしない）
- Form fields: 通常・focus・error・select・textarea

### 5. `04 Mobile 390` — モバイル優先の調整

- Heroの装飾は一層減らし、見出しとCTAのコントラストを確保する。
- sticky演出は通常スクロールに落とせる設計にする。
- 2カラムのプロダクト比較、統計、カード群は一列にする。
- `prefers-reduced-motion` 用に、すべてのスクロール演出の静止状態を定義する。

## 書き出し仕様

| 種別 | 形式 | サイズ・運用 |
| --- | --- | --- |
| ロゴ・アイコン・線画 | SVG | `public/assets/` に配置。青／白の両方を用意。 |
| 光・グラデーション・粒子の背景 | WebP または PNG | Desktop: 2880px幅、Mobile: 1170px幅。CSS上で位置を変えられる余白を含める。 |
| OG画像 | PNG | 1200 × 630。ロゴ、主メッセージ、抽象花のみ。 |
| UIモック | Figma component + 必要時SVG/PNG | ダミーデータのみ。実際のSentra画面や個人データを画像化しない。 |
| モーション | Figma prototype + 実装注記 | 動画素材を先に固定せず、Framer Motion/CSSで再現可能な変形に限定する。 |

## 実装の進め方

1. Figmaで `00 Foundations`、`01 Brand Assets`、Heroの3状態を作る。
2. Heroを `BLESC-website` に実装し、デスクトップ／モバイル／reduced-motionで確認する。
3. How it works と Product の2シーンを作り、sticky・パララックスの性能を検証する。
4. 残りのセクションを既存コピーに合わせて展開し、フォームとCTAの可読性を最終確認する。
5. 書き出したアセットをコミット前に最適化し、不要な巨大PNGや実データを含まないことを確認する。

## 完了条件

- Figmaに Foundations、Brand Assets、Desktop、Mobile、Components の各ページがある。
- Desktop は全シーンの開始／中間／終了状態、Mobile は静止状態を確認できる。
- 現行のコピー、ロゴ、公開ポリシーと矛盾しない。
- 最初の実装対象（Hero）に必要な書き出し物と、動きの数値注記が揃っている。
- 未成年や相談内容を特定できる素材を含まない。

## 次に必要なもの

Figmaへ実際に作成する段階では、新規Figmaファイルを作成するか、編集してよい既存FigmaファイルのURLを指定する。そのファイルにこの計画の `00 Foundations` から順に追加する。
